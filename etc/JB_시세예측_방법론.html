적정시세는 현재 가격을 정확히 아는 것입니다. 하지만 심사에서 진짜 필요한 건 대출 실행 이후의 시세입니다.

현재 예측 방식의 한계

현재 코드는 단순 선형 예측을 사용합니다. 직관적이지만 근본적인 한계가 있습니다.
// 현재 예측 로직
const isUptrend = latestNaverPrice > latestMolitPrice;
const annualRate = isUptrend ? 0.05 : -0.05;

// 매달 동일 비율로 직선 증가/감소
predictedPrice = latestJbPrice
  * Math.pow(1 + monthlyRate, m);

// 불확실성(신뢰구간) 없음
// 상승 또는 하락 둘 중 하나
코드 자체는 단순 선형 ±5% 직선
→ 심사 판단 근거로 쓰기엔 너무 단순

호가 vs 실거래만 보고 상승/하락 결정

호가가 실거래보다 높다고 항상 상승장이 아님
호가는 항상 희망가 포함 → 구조적 편향 존재

±5% 근거 없음

어떤 단지든 동일하게 5% 적용
실제 단지별 변동성은 천차만별

직선 예측의 한계

부동산은 직선으로 오르내리지 않음
계절성, 변곡점, 둔화 구간 모두 무시

불확실성 표현 불가

단일값만 출력 → "이 예측이 얼마나 믿을 만한가"
전달 불가 → 심사 근거로 활용 어려움

AutoARIMA 가 하는 일 : 단계별로 보면

✋ 현업 판단 필요
1

과거 시세 데이터 입력

단지 + 평형 기준 월별 적정시세 이력을 입력합니다. 최소 12개월, 이상적으로는 24~36개월 데이터를 사용합니다.

데이터 준비
2

패턴 자동 탐색

"이 데이터는 몇 달 전 패턴을 따르는가", "계절성이 있는가", "추세가 있는가"를 수백 가지 조합으로 자동 테스트합니다.

자동 최적화
3

최적 공식 선택

AIC(정보기준) 지표로 과적합 없이 가장 잘 설명하는 공식을 자동 채택합니다. 사람이 개입하지 않습니다.

자동 선택
4

예측값 + 신뢰구간 출력

대출기간(예: 12개월) 동안의 예측 시세와 함께 상한·하한 신뢰구간을 함께 출력합니다. 이 하한선이 심사의 핵심 기준이 됩니다.

결과 출력
✋ 현업 판단 필요 슬라이드 5
  • 예측 기간 설정 — 기본값 12개월이 심사 기준상 적절한지 확인. 단기 대출(6개월)과 장기 대출(24개월)에 다르게 적용할지 여부도 결정 필요

신뢰구간 : 예측의 불확실성을 숫자로 표현한다

✋ 현업 판단 필요
단일 예측값 하나보다 "어느 범위 안에 있을 가능성이 90%"가 심사에 훨씬 유용합니다.

예측 결과 예시 (12개월 예측)

📈 예측값 (중앙선)

가장 가능성 높은 미래 시세. 참고값으로 활용하지만 심사 기준으로 직접 쓰지 않음

🔵 신뢰구간 (90%)

실제 시세가 이 범위 안에 있을 가능성 90%. 범위가 넓을수록 예측 불확실성 높음을 의미

🔴 하한선 → 심사 핵심 기준

가장 나쁜 시나리오에서의 시세. 이 하한선 기준으로 LTV를 계산해야 보수적 심사 가능

✅ 실무 활용 원칙

하한선 기준 LTV ≤ 70% → 대출 실행 안전
70~85% → 조건부 / 85% 초과 → 한도 재검토

✋ 현업 판단 필요 슬라이드 6
  • 신뢰구간 %를 90%로 할지 95%로 할지 결정 필요 — 90%는 상대적으로 좁은 구간(덜 보수적), 95%는 넓은 구간(더 보수적). 심사 보수성 기준에 따라 선택

예측 시세를 LTV 심사에 적용하는 방법

✋ 현업 판단 필요
같은 대출 조건이라도 예측 시세 기준 LTV까지 함께 보면 리스크가 다르게 보입니다.

✋ 현업 판단 필요 슬라이드 7
  • LTV 임계값 기준 확정 — 현재 예시는 70%(안전) / 85%(위험) 기준. 사내 심사 기준과 일치하는지 확인 필요
  • 하한선 기준 적용 범위 — 모든 대출에 동일 적용할지, 대출 금액·기간·지역에 따라 기준을 달리 할지 결정 필요
